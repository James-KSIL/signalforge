# SignalForge - Week 1 State (Wedge Established)

## Current Form

SignalForge is a deterministic validation gate that enforces whether a codebase satisfies a declared intent contract.

It does not generate code.
It does not infer intent.
It strictly validates:

"Does the current implementation satisfy what was explicitly declared?"

Current form: local CLI wedge.

## Core Mechanism

### Input (Single Source of Truth)

File: `signalforge.yaml`

Structure:

```yaml
intent: "<one sentence describing system intent>"
constraints:
  - "<explicit deterministic rule>"
scope: "<directory to validate>"
```

Key properties:

- explicit
- versionable
- human-readable
- no hidden inference

### Execution

Command:

```bash
npx tsx scripts/sf-validate.ts
```

Output (clean deterministic):

- `SAFE`
- `NOT SAFE`

Failure shape:

```text
NOT SAFE

Violations:
- <reason>
```

No banners, no ambiguity.

## Deterministic Rules Implemented

### Base Rules (always active)

1. Scope must exist.
- Fail if directory does not exist.

2. Scope must contain files.
- Fail if directory exists but is empty.

### Constraint-Driven Rules (exact string match only)

Rules activate only if explicitly declared in `constraints`.

TypeScript constraint string:

- `scope must contain TypeScript files`

Behavior:

- checks for `.ts` or `.tsx`
- fail if none found

Violation text:

- `Declared scope contains no TypeScript files: <scope>`

Python constraint string:

- `scope must contain Python files`

Behavior:

- checks for `.py`
- fail if none found

Violation text:

- `Declared scope contains no Python files: <scope>`

Design invariants:

- no NLP
- no fuzzy matching
- no rule generalization
- no inference

Everything is exact, explicit, deterministic.

## Proven Behavior (Proof Pack)

Validated scenarios:

1. Missing scope -> `NOT SAFE`
- `Declared scope does not exist`

2. Empty scope -> `NOT SAFE`
- `Declared scope contains no files`

3. Valid TypeScript scope -> `SAFE`

4. Python constraint failure -> `NOT SAFE`
- `Declared scope contains no Python files`

Artifact:

- `artifacts/proof-packs/sf-validate-proof-pack.md`

## Architectural Position

Current system is primitive but correct.

It proves:

- intent can be declared simply
- validation can be deterministic
- output can be binary and trustworthy
- rules can be enforced without AI

## Why This Matters

SignalForge creates a contract -> validation -> enforcement boundary.

Comparison:

- linting checks syntax
- tests check behavior
- SignalForge checks intent alignment

## Known and Accepted Week 1 Limits

- constraints are hardcoded
- local only, no repo integration
- no PR visibility
- manual execution required
- limited rule set
